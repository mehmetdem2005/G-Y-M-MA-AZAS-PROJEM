BÖLÜM 2: PROTOKOL, POLİTİKA VE PROSEDÜR ÜÇGENİNİN TAM TANIMI

2.1. Protokol Katmanı (Zorunlu İletişim Şartnamesi)

Her ajanın dış dünyaya açılan tek kapısı protokollerdir. Ajanlar birbirlerini sadece protokol tanımları üzerinden tanır.

Genel Protokol Başlıkları (Her Mesajda Bulunur):

```yaml
protocol_id: string          # Örn: "Code_Generation_v2"
message_id: UUIDv7
correlation_id: UUIDv7
sender:
  agent_id: string
  role: string               # "orchestrator", "coder", "guardian", "memory", "policy"
  capability_version: string # Ajanın yetenek seti versiyonu
recipient:
  agent_id: string | "*"     # Hedef ajan veya broadcast
  required_skill: string     # Alıcıda bulunması gereken skill (opsiyonel)
timestamp: RFC3339
ttl_seconds: int
security:
  jwt_token: string          # ZeroTrustVerifier tarafından imzalanmış
  integrity_hash: string     # SHA256(payload + timestamp)
policy_assertions:           # Göndericinin uyduğunu beyan ettiği politikalar
  - policy_id: "P-CODE-01"
    evidence_ref: string
payload: object              # Protokole özgü veri
```

Önemli Protokol ID'leri ve Payload Şemaları (Örnek):

Protokol ID Amaç Zorunlu Payload Alanları
Code_Generation_v2 Kod üretimi talebi language, specification, context_files[], constraints[]
SAST_Scan_Request_v2 Güvenlik taraması talebi code_blob, language, entrypoint, cwe_filter[]
A2A_Negotiation_v2 Ajanlar arası müzakere proposal, fsm_state, contest_reason?, waiver_id?
Preview_Update_v1 Canlı önizleme güncellemesi component_id, html, css, js
Policy_Evaluation_Request_v1 Politika karar talebi subject, action, resource, context
Memory_Store_v1 Belleğe yazma talebi memory_type (vector/graph/procedural), content, metadata
Tool_Execution_v1 Ajanın bir tool çağırması tool_name, parameters, caller_agent
Hook_Trigger_v1 Bir hook'un tetiklendiğini bildirme hook_name, event_data, source_agent

2.3. Prosedür Katmanı (SOP - Standart Operasyon Prosedürleri)

Prosedürler, belirli bir olay karşısında sistemin izleyeceği adım adım iş akışlarıdır. ProcedureEngine tarafından yürütülür.

Örnek Prosedür: SEC_INCIDENT_RESPONSE_V1

```yaml
procedure_id: "SEC_INCIDENT_RESPONSE_V1"
trigger:
  event: "policy_violation"
  condition: "policy.severity == 'CRITICAL' and policy.category == 'security'"
steps:
  - step_id: 1
    action: "notify"
    target: "SecurityScanner"
    message: "Kritik güvenlik ihlali tespit edildi. İnceleme başlatılıyor."
    timeout_seconds: 10

  - step_id: 2
    action: "quarantine"
    target: "PipelineOrchestrator"
    parameters:
      scope: "affected_component"
      duration_minutes: 30
    on_failure: "ESCALATE"

  - step_id: 3
    action: "collect_evidence"
    target: "AuditTrailWORM"
    parameters:
      include_memory_dump: true
      include_code_snippet: true

  - step_id: 4
    action: "llm_analysis"
    target: "SecurityScanner"
    prompt_template: "Aşağıdaki kod parçası neden {policy_id} ihlaline neden oldu? Kısa bir özet geç: {code_snippet}"
    output_var: "incident_summary"

  - step_id: 5
    action: "request_approval"
    target: "human_operator"
    message: "Kritik güvenlik ihlali: {{incident_summary}}. Devam etmek için onay bekleniyor."
    options: ["Kodu Düzelt", "Muafiyet Tanı (Waive)", "Build'i İptal Et"]

  - step_id: 6
    action: "execute_decision"
    switch: "{{approval_result}}"
    cases:
      "Kodu Düzelt": 
        - action: "assign_task"
          target: "BackendCoder"
          task: "fix_security_issue"
      "Muafiyet Tanı (Waive)":
        - action: "create_waiver"
          target: "PolicyEnforcer"
      "Build'i İptal Et":
        - action: "abort_pipeline"
          target: "PipelineOrchestrator"
```

3.1. Hook Sistemi

Hook Adı Tetiklenme Anı Kullanım Amacı
before_llm_call LLM çağrısından hemen önce Prompt'a ek bağlam ekleme, PII filtreleme (P-PRIV-01)
after_llm_call LLM yanıtı alındıktan sonra Yanıtı JSON Schema ile doğrulama (P-LLM-02), hallüsinasyon kontrolü
