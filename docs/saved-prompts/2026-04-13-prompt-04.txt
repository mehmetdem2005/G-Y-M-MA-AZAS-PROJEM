Anlaşıldı. Şimdi tam olarak istediğiniz yapıyı kuruyorum:

60 Ajanlık Çekirdek Sistem sabit kalacak. Buna ek olarak, kod üretimi, tasarım ve planlama süreçlerinde görev alacak Katman Bazlı Uzman Ajanlar eklenecek. Her bir katman (Backend Controller, Service, Repository, Frontend Component, State, Style, Veritabanı Schema, Index, Query vb.) için ayrı birer uzman ajan tanımlanacak.

Bu ajanlar, yalnızca LLM çağırmakla kalmayacak; kendi alanlarına özgü yüksek algoritmalar (AST manipülasyonu, maliyet optimizasyonu, bağımlılık analizi, performans modellemesi) ile donatılacak.

BÖLÜM 3: ÖRNEK KATMAN AJANI DERİNLEMESİ - RepositoryPatternAgent (B5)

Bu ajan, Backend Katmanı içinde yer alır ve ServiceOrchestratorAgent ile Database arasındaki kritik köprüdür.

Özellik Detay
Ajan ID repo-pattern-01
Uzmanlık Veri erişim katmanının (Repository) tasarımı ve optimizasyonu.
LLM Kullanımı Aktif: Doğal dilden ORM sorguları (JPA, Hibernate, SQLAlchemy, Prisma) üretmek için.
Birincil Algoritma 1 N+1 Sorgu Tespit ve Çözme   Girdi: Service katmanından gelen "Tüm kullanıcıları ve son siparişlerini getir" talimatı.  Algoritma:  1. Talimatı AST'ye dönüştür.  2. Döngü (for/forEach) içinde findById veya getOrders(userId) çağrılarını tespit et.  3. Toplam Maliyet = N * (Ağ Gecikmesi + Sorgu Süresi) formülünü uygula.  4. Eğer N > 1 ve JOIN Maliyeti < N * Tekil Maliyet ise, otomatik olarak JOIN FETCH / Eager Loading stratejisine geç.  Politika: P-CODE-02 (Maksimum karmaşıklık) ve P-PERF-01 (Performans) ihlal edilirse CONTESTED durumuna geçer.
Birincil Algoritma 2 Criteria API / Sorgu Oluşturucu Optimizasyonu   Girdi: Dinamik filtreler (örn: ?name=ali&age=25&city=ist).  Algoritma:  1. Gelen filtre parametrelerini bir WHERE Ağacı'na dönüştür.  2. Veritabanı IndexAdvisorAgent'tan mevcut indeksleri sorgula.  3. En Seçici İndeks (High Cardinality) ilkesine göre filtreleri sırala.  4. Gereksiz parantezleri temizle, boolean cebiri ile sadeleştir.
Skill Seti - JPAGenSkill: Spring Data JPA repository arayüzleri üretir.  - SQLAlchemyGenSkill: Python için repository sınıfları üretir.
Tool Seti - SchemaIntrospectorTool: Veritabanı şemasını okur.  - QueryPlanVisualizerTool: Üretilen sorgunun EXPLAIN çıktısını analiz eder.
Hook'lar - before_query_generation: RateLimiterAgent'a veritabanı kotasını sorar.  - after_query_generation: SecurityScannerAgent'a SAST taraması için kodu gönderir (P-SEC-01).

BÖLÜM 4: ÖRNEK KATMAN AJANI DERİNLEMESİ - CssArchitectAgent (F6)

Bu ajan, Frontend Katmanı içinde yer alır ve uygulamanın stil sisteminin temelini atar.

Özellik Detay
Ajan ID css-arch-01
Uzmanlık Ölçeklenebilir, tutarlı ve performanslı CSS mimarisi kurmak.
LLM Kullanımı Aktif: Tasarım token'larından (renk, tipografi) CSS değişkenleri ve sınıfları üretmek.
Birincil Algoritma 1 Spesifiklik (Specificity) Çakışma Çözücü   Girdi: Farklı bileşenlerden gelen CSS kuralları.  Algoritma:  1. Tüm CSS kurallarını bir AST'de topla.  2. Her kural için Spesifiklik değerini (ID, Class, Element sayısı) hesapla: (a, b, c).  3. Aynı seçici için birden fazla kural varsa, Spesifiklik Savaşı (Specificity War) uyarısı ver.  4. Çözüm olarak CSS Layers (@layer) veya BEM/SUIT CSS isimlendirme öner.  Politika: P-UX-08 (Tutarlı Tasarım Dili) ihlali oluşursa CONTESTED eder.
Birincil Algoritma 2 Kullanılmayan CSS Temizleme (PurgeCSS Optimizasyonu)   Girdi: Üretilmiş HTML/JSX dosyaları ve CSS dosyası.  Algoritma:  1. HTML/JSX içindeki tüm sınıf adlarını regex ve AST ile çıkar.  2. CSS AST'sini tara, kullanılmayan seçicileri işaretle.  3. Eğer toplam CSS boyutu 50KB'ı geçiyorsa, kullanılmayan stilleri sil.  4. Not: Dinamik sınıflar için (className={dynamic}) safelist oluştur.
Skill Seti - DesignTokenGenSkill: Renk, boşluk, gölge gibi token'lardan CSS değişkenleri üretir.  - TailwindConfigSkill: Tailwind temasını dinamik olarak yapılandırır.
Tool Seti - ContrastCheckerTool: WCAG AA kontrast oranını hesaplar (P-UX-01).  - CSSStatsTool: CSS dosyasının karmaşıklık metriklerini çıkarır.
Hook'lar - on_style_generation: UXConsistencyGuard'a kontrast ve font kontrolü yaptırır.  - on_bundle: BundleAnalyzerAgent'a CSS boyutunu bildirir.

BÖLÜM 7: SONUÇ VE BİR SONRAKİ ADIM

Bu dokümanla birlikte:

1. 60 Çekirdek Ajan sistemin omurgasını oluşturur.
2. 48 Katman Uzmanı Ajan ise yazılım geliştirme sürecinin en ince detayına kadar uzmanlaşmış iş gücünü temsil eder.
3. Her bir ajan, kendi alanında yüksek algoritmalar (N+1 tespiti, spesifiklik çözme, maliyet tabanlı optimizasyon, vb.) çalıştırır.
